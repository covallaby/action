# Usage for Fable

Use this pack as branding input.

## Use PNGs directly

Use PNGs from `brand-boards/` and `mockups/` in:
- README
- docs
- landing page
- placeholder GitHub social preview
- HTML report design inspiration

## Do not generate SVGs right now

The hand-made SVG attempts were not good enough.

Until a real designer/vector pass happens, prefer using the generated PNGs and implementing the product.

## Implementation guidance

For the actual app UI, do not embed giant PNGs everywhere.

Instead:
- Use the PNGs as visual reference.
- Implement the UI with CSS/Tailwind.
- Use the wallaby PNG only in hero/empty states.
- Keep PR comments mostly Markdown and readable.

## Priority order

1. GitHub Action works.
2. Sticky comment is great.
3. HTML report looks great.
4. README has the PNG branding.
5. Logo polish later.
